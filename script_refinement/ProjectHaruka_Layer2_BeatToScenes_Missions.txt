# Project Haruka — Layer 2: Beat → Scenes / Missions (TXT)

Generated: 2026-01-15 (Asia/Tokyo)

Source: Beat Sheet Prompt 5 (B01–B26) + linked AtomIDs per beat.

Goal: Turn each beat into playable scene nodes you can map to quests, missions, cutscenes, and level beats.

Convention: SceneID = Bxx.Syy

Legend (Core Hook): Explore / Social / Combat / Puzzle / Stealth / Boss / Choice / Dream


============================================================
GLOBAL DESIGN NOTES (keep in mind while implementing)
============================================================
- Use “Info Tokens” per scene: what the audience learns, what the protagonist learns, what the team learns.
- “Social Evaluation” beats should have a visible meter + rumor posts + NPC behavior shifts (even if abstracted).
- Power/Cost beats: each time player uses “big power” show a small permanent trade-off (UI corruption, memory glitch, relationship fracture, map scar).
- Act 3 must feel like: victory momentum → moral dead-end → structure collapse (偽高潮→地震).
- Act 4 must feel like: truth earthquake → new-world rules → cost settlement (不是爽快勝利).
