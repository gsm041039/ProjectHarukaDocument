## 一、整合狀態總覽

### ✅ 已整合內容

#### 1. 哲學框架（已整合至 14_Alpha_Beta_Narrative_Mechanics.md）

**核心定義**：
- **Alpha線** = 事件層面的真相 (Truth of Events / What actually happened)
- **Beta線** = 態度/意義層面的真相 (Truth of Attitude / What it meant)
- **結局方向** = 融合/整合 (Integration)，而非二選一

**主題取向**：
- 愛自己 (self-care / self-acceptance)
- 核心轉變：「我唔想痛所以用魔法」→「我唔用魔法都要照顧自己」

**對照關係**：
| Alpha線（命運） | Beta線（態度） |
|----------------|----------------|
| 無意義的腐爛 | 有意義的燃燒 |
| 被動的受害者 | 主動的殉道者 |
| 窩囊無尊嚴的痛苦 | 壯烈有尊嚴的犧牲 |

#### 2. 核心機制（已整合至 14_Alpha_Beta_Narrative_Mechanics.md）

**同步機制**：
- 體感比例：戒斷60%（止痛貼被撕開）+ 真相40%（Alpha碎片衝入）
- 觸發條件：變身、使用魔法、情緒超載
- 本質：情緒超載令止痛需求暴增，魔法是最快的投藥方式

**命名機制**：
- 將「唔舒服」轉化為「我係怕X / 我係羞恥X」
- 命名即掌控，賦予混亂經驗以結構和意義

**共享印證機制**：
- 以共享為主軸（多人多路徑共鳴一致）
- 少量證據加固（物件、地標、短句）
- 避免「用魔法才知道真相」的單一通道

**屍骸化機制**：
- 非單一因果（不是「因為你冇面對Alpha」的直線因果）
- 修正力/整合陰影隱喻
- 「唔面對」是風險倍增器，而非唯一成因

#### 3. 角色Alpha線設定（已整合至 01_Character_Background_Story.md）

**朱音**：
- Alpha：吸毒/流浪街頭/孤獨死於垃圾堆
- Beta：吞噬敵人保護同伴
- 轉變：從「索取」（麻醉自己）到「承擔」（保護他人）
- 主題：自私vs無私、封閉自我vs開放他人

**操**：
- Alpha：被父親強制改造性別 + 精神病院囚禁
- Beta：用魔法維持女性身體 + 與父親的力量平衡
- 主題：身體自主權、被定義vs自我定義
- 魔法本質：「線」與「縫合」源於父親手術刀，但刀柄在自己手上
- **已移除**：聲音違和感（根據用戶指示）

**晴香**：
- Alpha：絕對無力的倖存者（眼睜睜看美夜子死）
- Beta：絕對全能的創世神（但帶有創世者原罪）
- 罪的轉移：為逃避「無力救人」之痛，選擇「折磨眾人」之罪
- 治癒：承認「有些事救不到」比「強行去救」需要更大勇氣

**美夜子**：
- 唯一Alpha已死的角色（其他人是「生不如死」）
- Alpha：突然中斷，留下遺憾
- Beta：借來的道別時光
- 任務：將晴香從「依賴姐姐的孩子」變成「能送走姐姐的大人」

**夕**：
- 定位：晴香的陰影 / Alpha線的代言人
- **已移除**：打破第四面牆的旁白者功能
- 核心：代表現實/殘酷/真相的一面

**凜**：
- 暫時跳過，待後續補充

#### 4. 心理整合框架（已整合至 06_Character_Psychology_Analysis.md）

**整合路徑**：
- Alpha創傷如何影響Beta人格
- 從被動受害者到主動選擇者的轉變
- 最終整合：帶著Alpha的傷痕，活出Beta的樣子

**治癒本質**：
- 不是消除痛苦，而是改變與痛苦的關係
- 不是忘記創傷，而是重新詮釋創傷的意義
- 不是回到過去，而是帶著傷痕繼續前行

## 二、v0.1 文檔核心內容

> **來源**：用戶提供的「Alpha / Beta 線敘事與遊戲機制設計文件 v0.1」

### 核心目標 (North Star)

用 Alpha/Beta 線作為主題級敘事手法，服務主題走向：
> 由「我唔想痛所以用魔法」→「我唔用魔法都要照顧自己」

（第一步：承認/看清；第二步：轉化/行動；兩者都需要勇氣）

### 基本定義

**1.1 Alpha 線（Truth of Events）**
- 定位：事件層面的真相（What actually happened / 命運層面）
- 重要性：觀眾最終要「相信 Alpha」，因為共享印證（多人多路徑共鳴一致）

**1.2 Beta 線（Truth of Attitude / Meaning）**
- 定位：態度/意義層面的真相（What it meant / 我點樣活／我點樣選擇）
- 結局方向：融合/整合（Integration），而唔係二選一

### 主題取向：愛（C）是什麼？

愛自己（可以涵蓋愛他人，但光譜更偏向 self-care / self-acceptance）

Beta 的「愛」唔係純浪漫或大愛，而係：
- 我肯照顧自己
- 我肯承認自己會痛
- 我唔需要靠止痛（魔法）先算活得落去

### 同步（Synchronization）感覺與比例

**感覺比例**：戒斷/不適 60% + 真相來襲 40%

**核心引擎**：
- 變身本身（某些角色）
- 使用魔法/戰鬥快感（某些角色）
- 總結：同步係「情緒超載」令止痛需求暴增，而魔法只係最快嘅投藥方式

### 世界規則：屍骸化、命運修正力

Rule 1 = C（榮格式整合陰影的隱喻 + 命運/修正力）

**核心機制**：
- 屍骸化唔係「因為你冇面對 Alpha」嘅單一因果
- 而係：存在一個更宏觀的「命運修正力／緋潮」機制
- 當角色長期以魔法作止痛、拒絕承認/命名/整合 → 自我結構不穩 → 修正力介入把人推向崩解
- 「唔面對」係風險倍增器，而唔係唯一成因

### 叙事手法：共享印證（Sharing Validation）做主軸

**共享印證用於**：場景設計 + 小情節
**核心理念**：以「共享」做主軸，再用少量「證據」加固
**共享形式**：各自見到的內容可以唔同，但「描述/錨點」要一致

### 魔法的立意矛盾

**問題 A**：用多魔法＝更易同步＝更易見真相 → 會唔會變相「心理上獎勵」使用魔法？
**問題 B**：若真相只能靠魔法同步知道 → 會唔會懲罰嗰啲「從頭到尾都唔用魔法」的人？

**暫時結論**：
- 唔應該令魔法成為唯一通往真相嘅門票
- 真相應該可以透過「共享印證」路徑逐步成立
- 魔法同步最多係「快但危險／噪音更大」的路徑，而唔係純獎勵

### 命名（Naming）與「儀式化行動」

**主軸**：命名（A）已定
- 命名＝由「唔舒服」變成「我係怕X／我係羞恥X」

**配合**：儀式的形式未定（但需要）
- 除咗命名（語言），最好有實際行動／儀式
- 參考 Persona 5「撕開面具」嗰種「身體化」象徵，但唔抄

### Alpha/Beta 與鏡、陰影的整合方向

**三者連繫**：
- 鏡
- 陰影
- Alpha/Beta 線敘事

**新遊戲機制（種子）**：
用太多魔法 → 陰影會彈出嚟 → 變成你敵人
（可能係修正力的表現、或人格陰影的具象化）

### 角色/美術落地：雙時間線元素混血

**可操作方向**：
角色（例如操）可以同時帶有兩條時間線元素：
- Beta：魔法少女形象
- Alpha：精神病服/束縛/皮帶約束等元素

**沿呢個方向 develop**：
- 唔止操，其他角色都可用「Alpha 紋理」滲入 Beta 外觀
- 變成一種「叙事手法＝美術語言＝機制提示」一體化

### 遊戲化展望（未定，但方向清楚）

**例子**：
- Alpha/Beta 可以變成可編程的遊戲機制
- 戰術性強制過量魔法以達到同步（短期手段）
- Alpha/Beta 設定下的特殊位移方式／關卡圖層玩法
- 鏡/陰影介面作為「同步/整合」的操作入口

**要求**：玩法要配合情緒設定（唔可以為機制而機制）

### 結局方向（Integration / Return to Alpha）

- 結局係融合/整合
- 若回到 Alpha，Beta 可以被理解成一種「心理治療式暴露療法／重建意義」
- 唔想落到「強制交功課/真理之門」太硬的設定

## 三、設計推導過程摘要

### 關鍵討論點

#### 1. 為何要深化Alpha線？

**問題**：如果 Alpha 線只係最後爆出嚟嘅設定，觀眾會覺得係「追加設定」。

**解決方案**：將 Alpha 線變成「慢性病 (Chronic Condition)」，貫穿成個故事。
- 她們越努力做魔法少女，Alpha 線死因就會越強烈地反噬她們
- 通過生理症狀、視覺滲漏、心理壓力等方式，讓Alpha線「活著」

#### 2. Alpha/Beta關係應該是什麼？

**問題**：如果兩條線都是「好慘」，就變成純粹賣慘。

**解決方案**：建立「鏡像倒置」關係：
- Alpha = 無意義的腐爛（被動受害）
- Beta = 有意義的燃燒（主動殉道）
- 關鍵是**性質不同**，而非**程度不同**

#### 3. 晴香的道德困境

**問題**：如果晴香只是「賦予尊嚴」，她就是大好人。

**解決方案**：加入道德灰色地帶：
- 夕的批判：「你只是將佢哋嘅慘死包裝成一場表演」
- Beta線的壯烈，可能只是對Alpha線慘狀的病態美化
- 這個困境沒有標準答案，留給觀眾思考

#### 4. 如何避免角色重疊？

**問題**：如果所有人Alpha線都死了，就跟美夜子重疊。

**解決方案**：差異化定位：
- 美夜子：唯一已死者（「借來的生命」）
- 朱音/操：生不如死（「無意義的腐爛」）
- 晴香：倖存者（「創世者原罪」）

#### 5. 凜的設定調整

**原討論內容**：凜因天真理想失敗→Beta過度冷血的修正

**用戶決策**：需要修改，暫時跳過
